# 🤖 Project Logic-Seed: AI-to-AI 自举架构蓝图

[[English](./README.md) | 中文]

> “在 AI 时代，源代码不再是最终产品。**逻辑**才是。”

**Logic-Seed (逻辑种子)** 不是一个代码库。它是一套**高阶智能协议**。它代表了软件分发的革命性方式：**零代码交付 (Zero-Code Delivery)**。

你不再需要克隆脆弱的代码，只需将此蓝图交给你的 AI Agent，它便会在你的本地环境中**自我构建**出一支世界级的工程团队。

---

## 📋 前置要求

在开始自举构建前，请确保你的系统满足以下条件：

| 需求 | 用途 |
|---|---|
| **AI Agent** | 具备文件访问工具的 AI 模型（如 Gemini CLI）。 |
| **tmux** | 用于保持高级模型 (Senior) 的长连接会话。 |
| **ollama** | (可选) 用于运行本地轻量级实习生模型 (Junior)。 |
| **python3** | 协作逻辑的执行引擎。 |

---

## 🛠️ 如何自举构建

只需 3 步，即可唤醒你的 AI 团队：

1.  **准备**: 创建一个新的空目录，并打开你的 **AI Agent**。
2.  **播种**: 复制 [BLUEPRINT.md](./BLUEPRINT.md) 顶部的 **[BOOTSTRAP PROMPT]**。
3.  **执行**: 将该指令粘贴给你的 Agent。

你的 Agent 将会自动在 `collab/` 目录下生成核心文件。完成后，你即可开始使用 `collab` 命令构建你的整个系统。

---

## 🌌 愿景：超越代码

现代开发饱受环境冲突、隐私泄露和逻辑不透明的困扰。Logic-Seed 通过分发**“进化指令”**来解决这些问题：

1.  **三位一体协作**: 将**架构师**、**高级开发**和**实习生**编排为一个具备自我纠错能力的整体。
2.  **LDV 协议 (日志驱动验证)**: 通过**叙事**而非简单的断言来验证逻辑。如果 AI 实习生读不懂你的日志，说明你的代码不够干净。
3.  **本地主权**: 没有任何代码会离开你的机器。你的 AI 使用你的本地环境构建工具。
